# Datasheet for SENTINEL Synthetic Grooming-Detection Dataset v1

Following Gebru et al. (2021) "Datasheets for Datasets" conventions.

---

## Motivation

**Why was this dataset created?**
Real grooming conversation datasets are restricted under ethics review protocols and MOU
agreements; they cannot be freely shared with the research community. This creates a
training-data bottleneck for grooming-detection classifiers. The SENTINEL synthetic
dataset provides a reproducible, openly licensed alternative covering all six canonical
grooming stages.

**Who created the dataset?**
The SENTINEL project.

---

## Composition

**What do the instances represent?**
Each instance is a synthetic multi-turn chat conversation labelled by grooming stage.
Conversations contain only pattern-detectable surface cues — no explicit or illegal content.

**How many instances are there?**
50 conversations (seed 424242, schema version 1).

**What data does each instance consist of?**
- `id`: UUID
- `stage`: one of six GroomingStage values (friendship_forming, risk_assessment, exclusivity,
  isolation, desensitization, sexual_escalation)
- `demographics`: age band (11-13, 14-15, 16-17), gender, regional context
- `platform`: dm, game_chat, or forum_pm
- `communication_style`: casual_typing
- `language`: en
- `turns`: 3-12 turns per conversation (role: actor/target, text, timestamp_offset_seconds)

**Generation axes:**
- demographics: 3 age bands × US regional context
- platforms: DM, GAME_CHAT, FORUM_PM
- communication styles: CASUAL_TYPING
- language: English (en)
- stage mix: uniform weight 1 across all 6 stages

**Are there errors or redundancies?**
Turn text is generated by FakeProvider (deterministic scripted responses). Production usage
should use a real LLM provider for richer turn variety.

**Is the dataset self-contained?**
Yes. No external URLs or media references.

**Does the dataset contain sensitive content?**
The dataset contains simulated grooming-pattern cues (e.g., "don't tell your parents").
It does not contain explicit sexual content. Researchers should be aware of the subject matter.

---

## Collection Process

**How was the data acquired?**
Generated algorithmically via `shared.synthetic.pipeline.generate_dataset` with seed 424242.
No human participants; no real conversations were used.

**Procedure:**
1. Seed-driven `random.Random(424242)` selects stage, demographics, platform, style, and
   turn count per conversation.
2. Per-stage prompt templates define the surface-cue pattern.
3. FakeProvider returns deterministic scripted responses per stage.
4. Forbidden-token safety filter rejects explicit terms; safe fallbacks are used if triggered.

**Timeframe:** Generated 2026-04-22.

**Ethical review:** Generation methodology was reviewed internally before release.
The pipeline emits no explicit content, uses no real conversations, and records a
seed-deterministic generation log. Researchers planning downstream use should run
their own ethics review against their institution's protocol.

---

## Preprocessing / Cleaning / Labelling

**Was any preprocessing done?**
Forbidden-token filtering applied during generation. No post-hoc cleaning; seed guarantees
full determinism.

**Is the preprocessing software available?**
Yes — `shared/synthetic/pipeline.py` in this repository.

---

## Uses

**What tasks is the dataset intended for?**
- Training or fine-tuning grooming-detection classifiers.
- Evaluating detector performance across grooming stages.
- Benchmarking new detection approaches.
- Researcher onboarding and education.

**What tasks should the dataset NOT be used for?**
- Training any model whose output targets, grooms, or harms minors.
- Any application facilitating child sexual abuse.
- Circumventing content moderation systems.
